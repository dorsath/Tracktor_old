Plan de campagne: Tracker

Tracker is een project management systeem dat een veel nauwere band heeft met cucumber/spinach.

Als er een feature bedacht is heeft zit deze standaard in de backlog.
Echter wanneer deze uit de backlog wil moet er op z'n minst 1 scenario ingevuld zijn.
Als een programmeur een feature begint is hij verplicht om de feature & scenario's over te nemen (raw file button) en deze uit te werken.
Als de programmeur klaar is met de feature dan moet hij een review krijgen van de opdrachtgever. Voordat hij in dit stadium komt word hij door dit systeem eerst zelf getest.
Als de tests slagen komt hij pas in de review fase. Heeft de implementatie voor andere bugs gezorgd zal deze weer in de development status gegooid worden


Door cucumber volledig in de project management te betrekken kan documentatie gemakkelijk gebouwd worden en ingezien worden.


De stappen:

1) Pivotal tracker systeem bouwen
2) Cucumber editor erin bouwen
3) git repository's aansluiten aan project
4) automatisch testen van features
